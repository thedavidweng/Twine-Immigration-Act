# Twin 项目一致性检查与修复计划

## 项目概述

这是一个基于 Twine 的互动叙事游戏，讲述加拿大华人的历史。根据 README.md，有三位作者：
- **1923, 2011-2023**: Makayla
- **1939-1947**: Ella  
- **1947-1960**: David

项目包含两个主要的 `.twee` 源文件和一个编译后的 `index.html`。

---

## 发现的一致性问题

### 1. 故事标题不一致

| 文件 | StoryTitle |
|------|------------|
| Stephen Wong.twee | "Stephen Wong" |
| Chinatown Memo.twee | "Chinatown Memo" |
| index.html | "Chinatown Memo" |

**问题**: 两个源文件使用不同的标题，但最终输出使用了 "Chinatown Memo"。

### 2. 图片 URL 不一致（最严重）

这是最明显的问题，反映了不同作者使用不同的资源托管方式：

| 章节 | Stephen Wong.twee | Chinatown Memo.twee |
|------|-------------------|---------------------|
| 1923 | dc.missuo.ru (外部) | github.com (David的资源) |
| 照片回忆 (2011) | dc.missuo.ru, p0.51img.ca | github.com |

**问题**: Stephen Wong.twee 使用了外部图片托管（可能是 Makayla 的版本），Chinatown Memo.twee 使用了 GitHub 原始链接（David 的版本）。

### 3. 结尾链接差异

- **Stephen Wong.twee** (第 64 行): `''The End''` → 结束，无后续链接
- **Chinatown Memo.twee** (第 64 行): `''The End''` → 链接到 `Citation` 页面

### 4. 链接名称微小差异

- `Stephen Wong.twee` 第 83 行: `[[ I followed Uncle Huang...]]` (有前导空格)
- `Chinatown Memo.twee` 第 86 行: `[[ I followed Uncle Huang...]]` (相同)

---

## 修复方案

### 步骤 1: 确定基准版本

经过详细对比分析，两个文件都有 **56 个 passages**，结构相同，但存在关键差异：

| 特性 | Stephen Wong.twee | Chinatown Memo.twee |
|------|-------------------|---------------------|
| StoryTitle | "Stephen Wong" | "Chinatown Memo" |
| 图片托管 | dc.missuo.ru, p0.51img.ca (外部) | github.com (统一) |
| 结尾 → Citation | 无链接 | 有链接 |
| index.html 来源 | 否 | 是 |

**结论**: 选择 `Chinatown Memo.twee` 作为基准版本，因为：
- 它是 index.html 的源文件
- 使用统一的 GitHub 图片托管
- 结尾有关闭引用功能

### 步骤 2: 修复 Stephen Wong.twee 以匹配基准版本

由于 Stephen Wong.twee 是旧版本/草稿，只需保留作为备份参考，无需修改。

**实际上：只需使用 Chinatown Memo.twee 作为最终成品**

### 步骤 3: 验证 index.html 是否需要重新编译

验证：index.html 使用的是 GitHub 图片链接（与 Chinatown Memo.twee 一致），无需重新编译。

**如果需要更新**：使用 Twine 将 Chinatown Memo.twee 编译为 index.html

---

## 结论

**最终成品是 `Chinatown Memo.twee`**（与 index.html 一致）

Stephen Wong.twee 是由另一位作者维护的旧版本，可以作为备份保留或删除。
